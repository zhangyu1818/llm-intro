# 双语 i18n 实施计划

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** 为「看懂大模型」站点加上 zh / en 双语支持，组件结构唯一、文案分语言 colocated，构建产出 `dist/index.html`（zh）与 `dist/en/index.html`（en）两份独立 HTML。

**Architecture:** 极薄 `LocaleProvider`（locale 由入口编译期写死）；每章 colocated `*.i18n.tsx` 文件存 `{ zh, en }` 字典，value 允许 ReactNode 保留富文本；Vite 多入口共享同一份组件树，仅 `main.zh.tsx` / `main.en.tsx` 分流语言；语言切换器走真跳转，保留 `#hash` 回到对应章节。

**Tech Stack:** React 19, Vite 8, TypeScript, Tailwind CSS v4。无新增运行时依赖。

**Spec:** `docs/superpowers/specs/2026-05-17-i18n-design.md`

**项目缺测试框架的约定**：每个 Task 的"验证"使用 `pnpm build`（含 `tsc -b`）、`pnpm lint`，以及必要的 `pnpm dev` 手动浏览检查。每次 commit 之前必须 `pnpm build` 通过。

- [ ] **Step 1：创建 `src/components/LangSwitcher.tsx`**

```tsx
import { useGlobal, useLocale } from '../i18n'

export function LangSwitcher() {
  const locale = useLocale()
  const g = useGlobal()
  const base = import.meta.env.BASE_URL
  const hash = typeof window === 'undefined' ? '' : window.location.hash
  const zhHref = `${base}${hash}`
  const enHref = `${base}en/${hash}`

  return (
    <div className="flex items-center gap-1 font-mono text-xs tracking-widest text-ink/60 uppercase">
      <a
        href={zhHref}
        aria-current={locale === 'zh' ? 'true' : undefined}
        className={locale === 'zh' ? 'text-ink' : 'transition hover:text-ink'}
      >
        {g.langSwitcher.zh}
      </a>
      <span className="text-ink/30">/</span>
      <a
        href={enHref}
        aria-current={locale === 'en' ? 'true' : undefined}
        className={locale === 'en' ? 'text-ink' : 'transition hover:text-ink'}
      >
        {g.langSwitcher.en}
      </a>
    </div>
  )
}
```
